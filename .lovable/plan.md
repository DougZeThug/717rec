

## Fix: Overly restrictive save validation and broken court numbering in useAutoScheduleSave

### Problems
1. **Standard mode** rejects any team appearing in multiple matches across the entire day, even at different timeslots — blocking legitimate double-headers.
2. **Dual match mode** hard-rejects rematches, but the scheduler treats rematch avoidance as a soft constraint — blocking valid "last resort" schedules.
3. **Court numbering** uses the global array index (`Court 1`, `Court 2`, ... `Court 11`...) instead of resetting per timeslot.

### Fix

**`src/hooks/useAutoSchedule/useAutoScheduleSave.ts`**

1. **Replace both validation branches (lines 47-137)** with a call to the central `validateMatchSchedule` from `@/utils/autoSchedule/validation.ts`. Check `result.isValid` and throw with a summary of errors if invalid. This aligns save-time validation with what the UI already shows.

2. **Fix court numbering (line 148)**: Group matches by timeslot, track a per-timeslot court counter, and assign `Court N` within each group.
